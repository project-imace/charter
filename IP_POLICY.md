# Intellectual Property Policy

Project IMACE (Integrated Modular Architecture for Cognitive Emulation) is an open research initiative committed to open use, responsible reuse, and clear attribution.

This policy establishes the principles governing ownership, attribution, redistribution, modification, and official recognition within the Project IMACE ecosystem.

---

## 1. Purpose

The purpose of this policy is to ensure that Project IMACE remains:

- open and accessible
- properly attributed
- protected against misrepresentation
- structurally independent in its official identity
- compatible with collaborative research and responsible reuse

This policy applies to all official Project IMACE repositories, projects, releases, and associated documentation unless explicitly stated otherwise.

---

## 2. Ownership of Official Project Identity

The name **Project IMACE**, its official releases, its structural documentation, and its designated branding are controlled by the Project IMACE coordination structure.

Only releases approved through Project IMACE governance may be represented as official Project IMACE output.

External contributors, collaborators, and affiliated groups may contribute to Project IMACE, but contribution does not transfer ownership of the project identity or official status.

---

## 3. Open Use and Redistribution

Project IMACE supports open use, modification, and redistribution of its code and research materials, subject to the applicable license of each repository or project.

Users may:

- use Project IMACE materials in research and development
- modify Project IMACE code for independent or derivative work
- redistribute Project IMACE materials in accordance with the applicable license
- build upon Project IMACE outputs in new projects

Open use is permitted, but open use does not remove the obligation to preserve attribution and prevent misrepresentation.

---

## 4. Attribution Requirements

Any substantial use, reuse, or redistribution of Project IMACE materials should preserve clear attribution to the Project IMACE project and, where relevant, to the specific contributing repository or release.

Attribution should remain visible in reasonable locations such as:

- source code headers or notices
- repository README files
- project documentation
- academic citations
- product documentation where applicable

Attribution should not be removed, obscured, or replaced in a manner that falsely suggests original authorship by another party.

---

## 5. No Misrepresentation

Derivative, forked, adapted, or redistributed versions of Project IMACE materials must not be presented as official Project IMACE releases unless explicitly approved by Project IMACE governance.

In particular, no party may:

- claim ownership of Project IMACE-originated work without attribution
- present a derivative project as the official Project IMACE project when it is not
- imply endorsement by Project IMACE without authorization
- remove or obscure the origin of Project IMACE materials in a way that misrepresents their source

Independent forks and derivatives are permitted where allowed by license, but they remain independent works unless formally recognized as official Project IMACE output.

---

## 6. Forks and Independent Derivatives

Forks, modifications, and independent derivatives are permitted under the applicable license.

However:

- they are not automatically official Project IMACE work
- they must preserve applicable notices and attribution requirements
- they must clearly distinguish themselves from official Project IMACE releases
- they must not use Project IMACE identity in a misleading manner

If a fork or derivative project uses Project IMACE materials, it should clearly state its relationship to the original project.

---

## 7. Project-Level Licensing

Each Project IMACE repository or project may define its own license, provided that the chosen license is consistent with the broader Project IMACE licensing policy.

Unless otherwise specified, Project IMACE projects should use open licenses that support:

- reuse
- modification
- redistribution
- attribution
- open research participation

Any project-specific exception must be explicitly documented and approved through the Project IMACE coordination structure.

---

## 8. Citation and Academic Use

Project IMACE encourages academic citation and research acknowledgment when its code, concepts, architecture, documentation, or research outputs are used in scholarly or technical work.

Where appropriate, users should cite:

- the Project IMACE project
- the relevant repository or release
- the authors or contributors associated with the work

A `CITATION.cff` file or equivalent citation metadata may be provided in project repositories to support academic reuse.

---

## 9. Branding and Name Use

The Project IMACE name, logo, and official branding are reserved for official project use unless otherwise authorized.

Use of the Project IMACE name in derivative projects, promotional material, or documentation must not create confusion about official status.

Any use of Project IMACE branding should respect:

- origin attribution
- official-release boundaries
- non-endorsement of independent derivatives unless explicitly approved

---

## 10. Contributor Responsibility

Contributors to Project IMACE retain ownership of their own original contributions unless otherwise agreed under the relevant project terms.

By contributing to Project IMACE, contributors agree that their work may be used, reviewed, integrated, and redistributed under the applicable project license and governance structure.

Contributors are responsible for ensuring that submitted work does not infringe third-party rights and does not introduce misattribution into the project.

---

## 11. Compliance and Review

The Project IMACE coordination structure may review issues related to:

- attribution
- official status
- reuse conditions
- branding
- derivative use
- licensing exceptions

Where concerns arise, the coordination structure may request clarification, correction, or removal of misleading claims.

---

## 12. Summary

In summary:

- Project IMACE materials are intended to remain open and usable.
- Attribution to Project IMACE must be preserved.
- Official Project IMACE status is controlled by Project IMACE governance.
- Forks and derivatives are allowed, but they are not official unless recognized.
- Project-level licenses may vary, but the broader Project IMACE identity must remain protected from misrepresentation.
