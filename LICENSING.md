<p align="center">
  Project IMACE (Integrated Modular Architecture for Cognitive Emulation) uses a repository-level licensing model.
</p>

> **Key principle:** different Project IMACE repositories may use different open-source licenses depending on the technical, research, and distribution requirements of the specific project.  
> Licensing must remain clear, open, attributed, and consistent with governance.

---

## At a Glance

| Layer | Default / Approved Model |
|---|---|
| **Code** | Apache 2.0, BSD 2-Clause, BSD 3-Clause, GPL v3.0, GPL v2.0, LGPL v3.0, MIT |
| **Documentation / Research Text** | CC BY-SA 4.0 |
| **Exceptions** | Repository-specific, explicit, and approved |
| **Authority** | Project IMACE coordination structure |

---

## 1. Purpose

This licensing policy exists to ensure that Project IMACE remains:

- open and reusable
- properly attributed
- legally clear
- compatible with collaborative research
- flexible enough to support different project needs
- protected against misrepresentation of origin and official status

---

## 2. General Licensing Model

Project IMACE does not require every repository to use the same license.

Instead, each repository must declare its own primary license clearly in the repository root.

The selected license must align with:

- the purpose of the repository
- the expected use of the code
- the openness requirements of the project
- the governance and attribution rules of Project IMACE

---

## 3. Approved Code Licenses

Project IMACE repositories may use one of the following licenses, depending on project context and approval:

| License | Typical Use |
|---|---|
| Apache License 2.0 | Broad reuse, adoption, clear legal framework |
| BSD 2-Clause License | Minimal permissive reuse |
| BSD 3-Clause License | Permissive reuse with name-use protection |
| GNU General Public License v3.0 | Strong copyleft, open derivatives |
| GNU General Public License v2.0 | Legacy copyleft compatibility |
| GNU Lesser General Public License v3.0 | Library-oriented copyleft |
| MIT License | Minimal permissive reuse |

The selected license must be stated clearly and must not be ambiguous.

---

## 4. Recommended Default

The recommended default license for many Project IMACE code repositories is:

**Apache License 2.0**

This is often suitable where the project aims to support broad reuse, redistribution, and adoption while preserving attribution and a clear legal framework.

However, the final choice remains repository-specific.

---

## 5. Copyleft and Stronger Share-Forward Licenses

Some Project IMACE repositories may use copyleft licenses such as:

- GNU General Public License v3.0
- GNU General Public License v2.0
- GNU Lesser General Public License v3.0

These licenses may be appropriate where the project intends to preserve openness across derivative use, redistribution, or library-level integration.

Where a copyleft license is used, the repository must clearly explain the license obligations and the expected reuse conditions.

---

## 6. Project-Specific Additional Permissions and Exceptions

For some repositories, the Project IMACE coordination structure may approve an additional permission, exception, or repository-specific license note to clarify how the code may be used, linked, distributed, or integrated.

Such additional permissions must be:

- explicit
- written into the repository documentation
- approved by Project IMACE governance
- consistent with the selected license
- free of ambiguity or hidden conditions

Any exception applies only to the specific repository in which it is documented.

---

## 7. Documentation and Research Text

Unless otherwise stated, project documentation, research notes, conceptual write-ups, and non-code materials in Project IMACE repositories should use:

**Creative Commons Attribution-ShareAlike 4.0 International (CC BY-SA 4.0)**

This license is suitable for materials intended to be reused, adapted, and redistributed with attribution and share-alike conditions.

If a repository contains both code and written research material, the repository should clearly distinguish which files are covered by the code license and which files are covered by the documentation license.

---

## 8. Attribution

All Project IMACE repositories must preserve attribution in accordance with their license and policy documents.

Attribution should not be removed, obscured, or replaced in a way that misrepresents the origin of the work.

Where applicable, repositories should include:

- `NOTICE`
- `CITATION.cff`
- copyright notices
- reference to Project IMACE as the originating project

---

## 9. Official Project Status

Licensing does not determine official status by itself.

Only releases approved through Project IMACE governance are considered official Project IMACE releases.

A fork, derivative, or modified version may be fully licensed and legally valid, but it is not automatically an official Project IMACE release.

---

## 10. Repository Requirements

Every Project IMACE repository must:

- declare its license clearly
- include the correct license file in the repository root
- state whether documentation follows CC BY-SA 4.0 or another approved documentation license
- preserve attribution and origin references
- document any approved additional permissions or exceptions

---

## 11. Governance Review

The Project IMACE coordination structure may review licensing decisions for:

- compatibility with project goals
- clarity of attribution
- openness and reuse
- repository-level exceptions
- compliance with official Project IMACE policy

Any proposed license exception or special permission must be documented and approved before it becomes part of the official policy.

## 13. Summary

In summary:

- Project IMACE allows different source code repositories to use different open-source licenses.
- Code repositories may use Apache 2.0, BSD, GPL, LGPL, or MIT, depending on project needs and approval.
- Documentation and research text should normally use CC BY-SA 4.0.
- Any special permission or exception must be explicit, documented, and approved.
- Attribution to Project IMACE must always be preserved.
- License choice does not override official governance or project identity.
